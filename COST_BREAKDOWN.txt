RISK ASSESSMENT: LOW

Potential Issues & Mitigation:
─────────────────────────────

1. CloudFront PriceClass Coverage
   Risk:     Reduced geographic coverage might affect distant users
   Reality:  PriceClass_100 covers 90%+ of users globally
   Monitor:  CloudFront metrics → Origin Shield metrics
   Rollback: Change back to PriceClass_200 (1 line, 3 min deploy)

2. 7-Day Cache TTL Staleness
   Risk:     Content updated but cached versions served for 7 days
   Reality:  Use free CloudFront invalidation (1000/month)
   Monitor:  Check if invalidations exceed 100/month
   Deploy:   aws cloudfront create-invalidation --paths "/*"
   Rollback: Adjust TTL down if needed

3. 404 Page Not Found
   Risk:     /404.html file missing breaks error handling
   Reality:  Easy to deploy file to S3 with rest of site
   Monitor:  Test 404 page after each deployment
   Deploy:   aws s3 cp 404.html s3://bucket/404.html

4. HTTP 404 Status Code
   Risk:     Returning HTTP 404 instead of 200 might affect SEO
   Reality:  Proper 404 codes are BETTER for SEO (industry standard)
   Monitor:  Google Search Console for crawl errors
   Benefit:  Search engines properly understand site structure

5. Disabled Versioning
   Risk:     Can't restore old versions from S3
   Reality:  Git history provides backup; versioning wastes storage
   Monitor:  No action needed (use git for version history)
   Rollback: Re-enable versioning anytime
